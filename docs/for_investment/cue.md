# Cue — невидимый AI-помощник

Tray-приложение, которое сидит в углу экрана и отвечает на вопросы за секунды. Часть экосистемы druz9.

> **One-liner:** «Stuck? Ask. Don't lose the moment.»

## Что это

Вызвал хоткеем `⌘⇧Space` → нарисовал прямоугольник → задал вопрос → получил ответ. Через 3 секунды окно исчезло. Никакой dock-иконки, никаких меню, никакой истории — только ответ под рукой.

Дополнительно — live-транскрипт встречи (Zoom/Meet/Teams) и auto-suggest при вопросах собеседника.

## Какую проблему решает

Когда разработчик застрял в legacy-коде, на сложном звонке, на финальном собесе — переключение в ChatGPT в браузере ломает фокус и забирает 2-3 минуты на копирование контекста. К моменту получения ответа момент уже упущен.

Cue убирает все шаги:

- **Скриншот вместо текста.** Любой код / диаграмма / интерфейс → прямо в prompt.
- **Live-транскрипт без записи.** На звонке слышит обе стороны, видит «что только что было».
- **RAG из CV / JD.** Перед собесом загрузил резюме и описание вакансии — ответы учитывают контекст.

## Ключевые фичи

- **Screenshot capture** — `⌘⇧S` / `⌘⇧A` для области экрана прямо в вопрос.
- **Voice input** — диктовка через Whisper.
- **macOS system audio capture** — встречи в Zoom/Meet/Teams транскрибируются в реальном времени без приватного MediaRecorder.
- **Auto-suggest pill** — на конце вопроса собеседника всплывает подсказка за 3-4 секунды.
- **RAG over CV/JD/URL** — `/documents` extract → embed → injected into prompt.
- **Невидимость при screen-share** — `setContentProtection(true)`. Zoom/Meet/Teams/OBS не видят окно.
- **Mock-block protocol** — во время strict mock-сессии на druz9.online Cue автоматически отказывает в подсказках. Сервер блокирует на уровне API (defense-in-depth), нельзя обойти модификацией клиента.
- **English Polish** — `⌃⇧L` читает буфер обмена → шлёт в Hone Writing-grader → stealth-окно с structured фидбэком (grammar / vocab / style / clarity) + one-click «Copy fix» → paste обратно в Slack/email/IDE-comment. Stealth то же что и compact (content-protection + always-on-top floating).

## Кому полезен и как используется

- **На финальном собесе** — voice + auto-suggest help. Один из ключевых сценариев целевой аудитории «middle/senior, готовится к собесу».
- **На коде / в IDE** — стрелка на кусок legacy кода, вопрос «что это делает» → ответ за 2 секунды без переключения окна.
- **На звонке с менеджером** — live-транскрипт + summary, чтобы потом не вспоминать.
- **При работе с документами** — приложил CV и JD до собеса, в течение часа Cue понимает контекст.

## Что Cue НЕ делает

- ❌ Не имеет dock-иконки, не имеет главного окна.
- ❌ Не пишет заметок, не ведёт стат — это работа Hone.
- ❌ Не запускается на mock-сессиях со strict-режимом — fairness gate.
- ❌ Не работает без Pro (главный платящий хук экосистемы).

## Дифференциация

- **Скорость.** Существующие AI-помощники (Cluely, Interview Coder) дают ответ за 5-10 секунд. Cue — за 2-4 за счёт VAD + boundary-aware streaming.
- **Контекст из всей экосистемы.** Cue знает Skill Atlas пользователя из druz9, видит его focus-сессии из Hone. Конкуренты — нет.
- **Defense-in-depth.** Mock-block — серверная блокировка, нельзя обойти. Это делает druz9 mock-результаты доверенной валютой; конкуренты не могут так же, потому что у них нет «арены, на которой нельзя жульничать».
- **Безопасность.** SSRF guards, prompt-injection sanitizers (`<<<USER_DOC>>>` delimiters), per-user daily token cap, 5 redis kill-switches. Это не game-changer для пользователя, но критично для масштабирования и B2B.

## Этичность и позиционирование

Stealth-фича существует для **законных сценариев**: работа в open-plan офисе без раздражения коллег, помощь себе на сложном этапе работы, запись своих встреч для собственного review. Mock-block протокол гарантирует, что для тренировочного слоя экосистемы Cue выключен в нужный момент. ToS прямо описывает ответственность пользователя за соответствие политикам работодателя.
