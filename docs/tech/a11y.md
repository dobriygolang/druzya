# a11y.md — WCAG audit + remediation

Live as of 2026-05-12 (Phase I launch readiness). Audited via `frontend/scripts/a11y-check.sh` (axe-core CLI through `npx`).

## Tooling

- **Script:** `frontend/scripts/a11y-check.sh` — runs `npx @axe-core/cli@4` against 13 main routes against tags `wcag2a,wcag2aa,wcag21aa,wcag2aaa,wcag21aaa,best-practice`.
- **No npm install:** uses npx so axe lives in npx-cache, not `node_modules`. Keeps prod bundle clean.
- **Run locally:** `bash frontend/scripts/a11y-check.sh` (requires dev server on :5173).
- **CI:** `BASE_URL=https://preview.druz9.online bash frontend/scripts/a11y-check.sh` after PR preview deploy.

## Severity ranking convention

| Sev | Meaning | Action |
|---|---|---|
| **S1** | Blocks keyboard / screen-reader users from accessing core functionality | Fix immediately |
| **S2** | AA failure on visible content (contrast, missing label) | Fix before launch |
| **S3** | AAA failure (low contrast, semantic gap) | Document, fix opportunistically |
| **S4** | Best-practice nit (e.g. landmark missing) | Defer |

| 1 | `frontend/src/pages/TaskBoardPage.tsx:675` | S2 | Drawer close X button missing `aria-label` and `type="button"` | Added `aria-label="Закрыть карточку"` + explicit `type="button"` |

| 4 | `frontend/src/components/AppShell.tsx:67` (NavItem) | S2 | Active nav link has no `aria-current="page"` — screen-readers can't announce current location | Added `aria-current={active ? 'page' : undefined}` |
| 5 | `frontend/index.html:2` | S4 | `<html>` missing forced `color-scheme: dark` → some UA controls render light | Added `class="dark" style="color-scheme: dark;"` on `<html>` |

Plus pre-existing strengths confirmed:
- `Button.tsx` ships with `focus-visible:ring` + `aria-busy` while loading.
- `primitives/Modal.tsx` + `primitives/Drawer.tsx` ship with `role="dialog" aria-modal="true" aria-labelledby` and `useFocusTrap`.
- `AppShell.tsx` icon-only buttons (search, palette, menu, avatar, close) all carry `aria-label`.
- `Avatar.tsx` ships with `aria-label` (when provided) + decorative `aria-hidden` initials.
- 45 component files already use `aria-label` (audited 2026-05-12).

## Keyboard navigation

Manual sweep passed:
- Tab cycles through TopNav → search → palette → bell → avatar → menu (mobile).
- Enter activates all buttons; modals trap focus via `useFocusTrap` hook (`lib/focus-trap.ts`).
- Esc closes modals/drawers (handled in `primitives/Modal.tsx` `useEffect`).
- All routes in App.tsx land on `<Suspense fallback={<RouteLoader/>}>` — no white flash that breaks focus.
